{% comment %} Writing archive — design-spec.md §6.4. One template behind /posts/, every tag page and every year page: two columns, compact entries separated by 1px rules, sidebar right (below on mobile). The spec's category filter pills were removed (George's call): on GitHub Pages with jekyll-paginate they could only ever filter the current page of 8, not the whole archive, which reads as broken. Category still shows on the homepage cards and each entry's metadata line; cross-archive filtering is served by the topic/year sidebar. Params: posts - the post list to render. paginator.posts on /posts/, or tag_posts on a tag page, or site.posts with `year` set. heading - page title, e.g. "Posts and Newsletters", "Strategy", "2025". year - optional "YYYY". Filters `posts` to that year. Passing the whole collection and filtering here avoids building an array in Liquid, which has no push filter. paginated - render the pager. Only /posts/ has one. {% endcomment %}
